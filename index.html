P1824 cow’s new house

des::

Farmer John建造了一个有N(2<=N<=100,000)个隔间的牛棚，这些隔间分布在一条直线上，坐标是x1,…,xN (0<=xi<=1,000,000,000)。

他的C(2<=C<=N)头牛不满于隔间的位置分布，它们为牛棚里其他的牛的存在而愤怒。为了防止牛之间的互相打斗，Farmer John想把这些牛安置在指定的隔间，所有牛中相邻两头的最近距离越大越好。那么，这个最大的最近距离是多少呢？

sol:看到 这个最大的最近距离 这种东西，应该能直接先想到二分答案。

定义变量ans，储存当前优解。定义闭区间[left, right]，代表程序当前正在此闭区间内寻找答案（寻找潜在的比ans更优的解）（与前两种方法不同的是，最优解不一定要属于该闭区间）。

令 $mid = (left + right)/2$

若$mid$为解，则$ans = max(ans, mid), left = mid + 1.$此时我们更新了最优解，同时在最优解的右侧寻找潜在的更有解。

若$mid$不为解，则$right = mid - 1$.$mid$不是解，因此我们在$mid$左边寻找更优解。

重复上述过程，直到$left > right$时跳出循环，$ans$即为最优解.

注意，当$left == right$时，也必须要在此区间内进行判断，因为当前还不能确定该区间内是否存在更优解。

AUOJ1367

des ：

给定长度为 $n$ 的序列。将其分为 $k$ 段，使得每一段的最大值-最小值 最小。

sol ：二分答案，主要是这个题面是我简化了的。

所以很明显可以想到二分答案。

其实二分答案的核心就是那个 $check()$ 。

我们每一次二分答案的值是 $mid$ .

那么对于 $check$ 函数所传的参数，我们只需要结合题目条件判断它是否可行就好（这里一般是 $\text{O}(n)$ 的。

那么这一道题就有几种情况。

  1. 当前枚举到的 $maxvalue-minvalue$ 大于 $mid$ 那么要重新分一次段。

  2. 当前所枚举出的总段数太大了。不可行。

  3. 枚举完之后 满足要求，当前解可行。

不过要稍微注意一下细节。

这里给出check的实现： 

bool valid(int value){
	int tot=0;//当前分了多少段 
	int maxv=-inf,minv=inf; 
	for(register int i=1;i<=n;++i){
		maxv=max(maxv,a[i]),minv=min(minv,a[i]);
		if(maxv-minv>value){
			++tot,maxv=minv=a[i];
		}//update
		if(tot>k){
			return false;
		}
	}
	++tot;//这里一定要记得加上！
	return tot<=k;
}

题面有一点长，大意是给定一堆直角三角形的坐标。求一条函数
$y=mid$ 使得它在 $[a,b]$ 之间的每一个点都有三角形覆盖。

求 $mid_{\text{max}}$ 。

考虑二分答案。因为题目要求的是那一条常数函数的解析式，直接二分它的那个$y$ 值。

为了避免实数二分。我们直接将所有读入的坐标倍增一下。

一倍就够了。（等腰RT三角形）

然后在check里面，我们维护一个差分数组。

看图：

比如说我们要维护 $B$ ，那么很明显在我们二分的值为 2 的时候，我们要维护一下差分数组的 $[2,6]$ 。

现在推广到一般情况。

若 $B(x,y)$ ，当前二分的值为 $value$ 。

那么我们做一条垂线 $BF$ （很明显$\tan \angle ABF =1$）

因为 $BF=y_B-value$

所以我们要维护的区间左端点 : $x-y+value$

右端点 : $x+y-value$

还有，因为二分答案的check总是有很多细节。所以每一次写的时候都要考虑全。

这里我们维护一次差分数组的大前提一定是 $y_B >= value$

（要不然您怎么算）

然后在看它的 $[a,b]$ （给出来的）的时候。我们还要看它和我们所有的 右端点 $x+y-value$ 的大小。如果不满足，那根本就没法覆盖。

然后我们 $\text{O}(n)$ 还原一下数组。

从 $[a,b]$ 扫一下看一下有没有 $0$ 即可（其实这个差分数组的本质就是 $i,value$ 这个点被几个三角形覆盖了，0就是没有）（另外这是要多次二分的，记得memset 0）。

记得二分开始的时候 $r$ 是所有三角形顶点的最大值，这样二分次数会少一些。

杂-对于DP的一点理解

发表于 2021-04-17 更新于 2021-05-22

关于DP

四要素

状态（重中之重）

都说推方程式 $\text{dp}$ 里最简单的，设计出好的状态才是难的。

一般我们都是设 $f_{i,j}$ 表示什么什么东西。

然后个人觉得状态就分了两个部分：阶段和条件。

  • 阶段

这种东西都是在循环最外层枚举的（比如 $\text{Floyd}$ 的 $k$），在状态里面体现出来也就是：

前i个,第i个,对于前i段……

当然你也可以有多个“阶段”，不过要分清楚顺序。

  • 条件/决策/……

这个其实就是状态的核心了。条件类型的话一般都是看题目条件而定。

比如： 用了j的空间（背包）,疲劳度为k（某道题）,走到第j个格子……

不过还有一种我将其称为“决策型”，为什么呢？

因为这种东西就是决定“状态”里的“决策”的，

比如 0表示休息，1表示跑,0表示max，1表示min 等等。

这些一般在决策需要分情况而且需要在状态里面体现的时候有用这种状态

方程

推得时候要结合着状态来，想清楚状态从哪里来，怎么转移。

然后有些题式需要分类讨论的，所以分情况就好。

不过注意，在找情况的时候要做到不重不漏，也就是不能少情况或者多情况。

有限制的转移可以压进状态，也可以判断解决转移。

转移的时候方程一般都是：$f_{i,j}=\operatorname{fuction}(f_{i,j},f_{……})$

为了保证“无后效性”，就要做到等式左边未知而右边全部已知。

如果你觉得方程是对的，但是右边有未知，就尝试用条件把它变成已知（或者再看看你的方程是不是推错了）。

（请把 $f$ 当做已知的东西用，在保证无后效性的情况下）

边界

初始值，根据状态来就好。

如果在外面初始化了就不用再循环里初始化，

当然如果你在循环里初始化那就没必要在外面写了（只要不漏就行）。

还有，memset的时候不要初始化double类型的东西。（其实个人觉得最好不用memset），也要注意它放在哪个位置（顺序）。

目标

也就是答案，根据你设的状态就行。

不过有的时候要在全部 $f$ 里面累加或者取最值。

也有可能循环出来了还要和原来的某种情况取最值（牛的旅行那道$\text{Floyd}$就是）。

题解 CF1504A 【Déjà Vu】

这场有那么一丢丢毒瘤。

不过A题还是一如既往的送分。

看一下题面有啥要求。

if there is no solution, output “NO”.

那么考虑怎么判无解（很明显无解情况少一点）。

这里要求你添加 'a' 之后不可以出现回文。

思考一下：只有原串全部是 'a' 才无解。

为什么？

如果你的字符串里有任何 'a' 以外的东西。

因为回文是需要像 a[i]=a[n-i+1] 一样的对应的。

可以直接把 a 放到这个不同字符对应的位置上。

所以一定有解。

然后考虑怎么构造。

上面那里已经说了，回文是要两个两个对应的。

找到第一个不是 'a' 的位置，在对应位置加一个 'a'即可。

If there are multiple solutions, you may print any.

这里多解任意输出即可，所以只要合法就行。

code:



#include<bits/stdc++.h>
using namespace std;

int T;

int main(){
	cin>>T;
	for(register int _=1;_<=T;++_){
		string s;
		cin>>s;int n=s.size();
		s=' '+s;
		int fuc=0;
		for(register int i=1;i<=n;++i){
			if(s[i]!='a'){
				fuc=0;break;
			}
			fuc=1;
		}
		if(fuc==1){
			cout<<"NO\n";
			continue;
		}//无解
		cout<<"YES\n";
		for(register int i=1;i<=n;++i){
			if(s[n-i+1]!='a'){//找到第一个位置
				for(register int j=1;j<i;++j){
					cout<<s[j];
				}cout<<"a";
				for(register int j=i;j<=n;++j){
					cout<<s[j];
				}cout<<endl;
				break;
			}
		}
	}
	return 0;
}

description:

对于完全图G，若有且仅有一棵最小生成树为T，则称完全图G是树T扩展出的。

给你一棵树T，找出T能扩展出的边权和最小的完全图G。

输入

第一行N表示树T的点数；

接下来N−1行三个整数Si,Ti,Di；描述一条边（Si,Ti）权值为Di；

保证输入数据构成一棵树。

输出

输出仅一个数，表示最小的完全图G的边权和。

solution:

因为给出的树是唯一最小生成树。

我们干脆在加边加边（然后……）的时候直接跑一个$\text{kruskal}$ 的“近似”算法。

考虑先把原有边排序，然后跑一个类似于mst的东西。

然后根据mst的推论。我们对于一条边 $(x,y)=val$ 考虑它们两个所在的不同集合。

然后对于另外的异于$(x,y)$ 的两个分别在$(x,y) $的集合之中的 $(u,v)$

我们考虑连另外的一条边，为了之后总值最小，我们连 $val+1$

然后这两个集合合并之前就要连 $|X|\times|Y|-1$条边，权值是 $val+1$。（X，Y是x,y所在的集合）

所以判完不连通之后才能合并。

然后这里是用生成树的性质做的。

所以我们只用考虑两个集合之间的边就好（可以想一下为什么）。

然后在过程里加一下就好。

那么代码：

//sum=原来的权值和
for(register int i=1;i<n;++i){
	int rx=root(e[i].x),ry=root(e[i].y);
	if(rx!=ry){
		sum+=(siz[rx]*siz[ry]-1)*(e[i].val+1);	
		pa[rx]=ry,siz[ry]+=siz[rx];
	}
}

这里有一种神奇的想法。

我们给所有白边都加上一个值$val$

那么有可能加到某个值，一条白边就被加入（踢出）mst了。

怎么确定这个值？

简单思考一下发现 $val$ 是满足单调性的。

所以考虑二分答案。

如果求出来的真实白边数 $real$ 大于$need$ ，考虑就要增加 $val$ 来减少白边。

反之亦然。

然后在$need \leq real$ 的时候更新答案就行了（因为这个时候才是可行的，可以自己证明一下。

这里给出代码：


#include<bits/stdc++.h>
using namespace std;

const int si=2e5+10;
int n,m,need;
int pa[si];

struct node{
	int x,y,val,ty;
	bool operator < (const node &u)const{
		if(val==u.val) return ty<u.ty;
		return val<u.val;
	}
}e[si];
int root(int x){
	if(pa[x]!=x){
		pa[x]=root(pa[x]);
	}
	return pa[x];
}
bool Union(int x,int y){
	int rx=root(x),ry=root(y);
	if(rx==ry){
		return false;
	}
	pa[rx]=ry;
	return true;
}

void kruskal(int &sum,int &real){
	sort(e+1,e+1+m);
	for(register int i=1;i<=n;++i){
		pa[i]=i;
	}
	for(register int i=1;i<=m;++i){
		if(Union(e[i].x,e[i].y)){
			sum+=e[i].val;
			real+=(int)(e[i].ty==0);
		}
	}
	return;
}

int main(){
	cin>>n>>m>>need;
	for(register int i=1;i<=m;++i){
		cin>>e[i].x>>e[i].y>>e[i].val>>e[i].ty;
		++e[i].x,++e[i].y;
	}
	int l=-114,r=114;
	int res=0;
	while(l<=r){
		int mid=(l+r)>>1;
		for(register int i=1;i<=m;++i){
			if(e[i].ty==0) e[i].val+=mid;
		}
		int sum=0,real=0;
		kruskal(sum,real);
		if(real>=need) res=sum-(need*mid),l=mid+1;
		else{
			r=mid-1;
		}
		for(register int i=1;i<=m;++i){
			if(e[i].ty==0) e[i].val-=mid;
		}
	}
	printf("%d\n",res);
	return 0;
}

using namespace std;

const int inf=114514333;
const int si=500;
int n;
double f[si][si];
struct node{
	int x,y;
}p[si];
double clac(int u,int v){
	return (double)sqrt((p[u].x-p[v].x)*(p[u].x-p[v].x)+(p[u].y-p[v].y)*(p[u].y-p[v].y));
}

void Floyd(){
	for(register int k=1;k<=n;++k){
		for(register int i=1;i<=n;++i){
			for(register int j=1;j<=n;++j){
				f[i][j]=min(f[i][j],f[i][k]+f[k][j]);
			}
		}
	}
}

double D[si];//直径（可以看做对于每个点的）

int main(){

		cin>>p[i].x>>p[i].y;

		for(register int j=1,o;j<=n;++j){
			scanf("%1d",&o);
			if(o==1) f[i][j]=clac(i,j);
			else if(i!=j) f[i][j]=(double)inf;
		}
	}
	Floyd();
	for(register int i=1;i<=n;++i){
		for(register int j=1;j<=n;++j){
			if(f[i][j]!=inf){
				D[i]=max(D[i],f[i][j]);
			}
			D1=max(D1,D[i]);
		}
	}

			if(f[i][j]==inf){
				double w=clac(i,j);
				D2=min(D2,D[i]+w+D[j]);
			}
		}
	}
	double ans=max(D1,D2); 
	printf("%.6lf\n",ans);

}

#include<bits/stdc++.h>

const int si=500+21;
int g[si][si];
int a[si];
struct node{
    int u,dep;
}q[si];
bool vis[si];
int n,m;
int bfs(int s){
	memset(vis,0,sizeof(vis));
	q[0]=(node){s,0};
	vis[s]=1;
	int head=0,tail=1;
	while(head<tail){
		node p=q[head++];
		if(p.u==n)return p.dep-1;
		int u=p.u;
		for(register int v=1;v<=si;v++){
			if(g[u][v]&&vis[v]==0){
				q[tail++]=(node){v,p.dep+1};
				vis[v]=1;
			}
		}
	}
	return -1;
}
int main(){

solution

这个题是一个比较典型的 小区间推大区间。

因为你会发现老张走路不一定是一条线。

也就是说正常的 线性$\text{dp}$ 是没有办法做这种题的（不能按从前往后或者从后往前推）。

那么这个时候就要联想到 区间$\text{dp}$ 这个“不按线性顺序”更新的东西

注意到这里老张是想“关灯”且尽量早关 , 也就是说他如果经过了一个灯他必定会关掉，不然再折回来就会浪费，不是最优的。

也就是说，他走路的顺序一定是类似于这种的（直线也包括在内）：

那么也就是说，对于一个区间,老张如果站在这个区间的端点的时候，那么他必定早已关完了这个区间所有的灯。

所以我们就可以从 $f[start][start]$ 这个最小的区间慢慢推到最大的 $f[1][n]$ 这个区间。

然后一个区间是有两个端点的，所以要用 $[0/1]$ 区分一下（因为不同的端点可能有不同的值，也就是方向的不同导致了结果的不同）。

至于怎么转移呢？

我们这里完全没有必要用一个 $k$ 去枚举上一个决策点了，因为我们推出来了这一条：

他如果经过了一个灯,那么他必定会关掉。

所以，比如在区间左端点的时候，那么就可以直接从 $l+1$ 这个位置转移过来。

所以每一次推大区间的时候都是 每次$+1$ 这样子推过去的。

当然你转移的时候也要考虑这个小区间是在左端点过来还是右端点过来。

（你也看到了，上图的路径是有重复的，所以不一定是离他最近的端点转移过来的）

然后考虑怎么计算花费，这里直接计算一下小区间的功耗然后加上扩展出来的（也就是端点的花费）就好了。

#include<bits/stdc++.h>
using namespace std;

const int si=60;
int a[si],b[si],sum[si];

这里你会发现和上一道关路灯基本是一样的。

也就是说Bessie的行走路线也是折线型不按顺序来的。

所以我们又一次地想到了区间 $\text{dp}$

但是这里有限制条件啊，咋办？

我们肯定没法用上一道关路灯的办法来设计状态。

因为Bessie经过一个教室不一定会交作业（因为还没下课）

那么我们就可以换个方式。

我们不是从小区间推大区间了。

我们从大区间推小区间。

也就是考虑 $f[i][j][0/1]$ 表示除了这一个区间以外，作业都交完了，马上去交i或者j处的作业的最值。

但是你会发现时间这东西烦人的很，会干扰我们的转移，比如出极大的数据（

所以干脆先排个序先，因为我们基本按照时间去DP肯定比之前的乱DP更优。

那么我们用时间排序之后，就可以利用现在时间单调的特性。

从 $f[1][n]$ 推到 每一个$f[i][i]$，然后和巴士站比较一下距离取最值就好了。

代码：

#include<bits/stdc++.h>
using namespace std;

const int si=2e3+10;
int n,h,b;
int f[si][si][2];

struct node{
	int x,t;
	bool operator <(const node &p)const{
		return x<p.x;
	}
}a[si];
int cald(int u,int v){
	return abs(a[u].x-a[v].x);
}

int main(){
	cin>>n>>h>>b;
	for(register int i=1;i<=n;i++){
		cin>>a[i].x>>a[i].t;
	}
	memset(f,0x3f3f3f,sizeof f);
	sort(a+1,a+n+1);//排序，贪心
	f[1][n][0]=max(a[1].x,a[1].t),f[1][n][1]=max(a[n].x,a[n].t);
	for(register int i=1;i<=n;i++){
		for(register int j=n;j>=1;j--){//倒着决策来从小推大
			f[i][j][0]=min(f[i][j][0],max(f[i-1][j][0]+cald(i-1,i),a[i].t)),f[i][j][0]=min(f[i][j][0],max(f[i][j+1][1]+cald(j+1,i),a[i].t));
			f[i][j][1]=min(f[i][j][1],max(f[i-1][j][0]+cald(i-1,j),a[j].t)),f[i][j][1]=min(f[i][j][1],max(f[i][j+1][1]+cald(j+1,j),a[j].t));
		}
	}
	int res=0x3f3f3f3f;
	for(register int i=1;i<=n;i++){
		res=min(res,
		min(f[i][i][0]+abs(b-a[i].x),
		f[i][i][1]+abs(b-a[i].x)));
	}//取最值
	cout<<res<<endl;
	return 0;
 }

发表于 2021-03-20

题解 CF1132F 【Clear the String】

区间Dp

看到这种可以区间区间删的东西。

直接就可以想到区间DP。

那么设 $f[i][j]$ 表示把 $[i,j) $ 删完 的最小代价。

考虑如何转移。

记得有一道 CQOI 的题，好像是叫涂色来着，和这道题的做法比较像。

那道题的经典思路就是判断 $s[j]=s[k]$ 是否为真，为真即表示这个区间 $[j,k)$ 可以涂上一种颜色。

看到这一道题，我联想到的就是雷同于涂色那道题的做法。

那么还是：判断 $s[j]=s[k]$ 是否为真。

不过这一道题是说直接删去，我们就可以想到，如果上式为真，那么合并这个区间的价值就直接是 $f[i][k]+f[k+1][j]$

反之，我们需要多删去 $s[j]$ 和 $s[k]$ 中的任意一个，那么就要花费 1 的代价。

所以不难搞出方程： f[i][j]=min(f[i][j],f[i][k]+f[k+1][j]-(int)(s[k]==s[j]));

还有，边界条件。

我们根据状态的定义，直接把所有的 $f[i][i]$ 都赋值为 1。

当然，如果我们在外面初始化了，在转移的时候，如果 $i=j$，我们直接 continue 掉就可以了。

代码：


#include<bits/stdc++.h>
using namespace std;

const int si=1003;
int n;
char s[si];
int f[si][si];

int main(){
    scanf("%d",&n);
    cin>>s+1;//把字符串长度加一
    memset(f,0x3f,sizeof(f));//先赋一个初值
    for(register int i=1;i<=n;++i){
        f[i][i]=1;
    }
    for(register int len=2;len<=n;++len){
        for(register int i=1;i+len-1<=n;++i){
            register int j=i+len-1;
            if(i==j){
                continue;
            }//i=j直接跳过
            else{
                for(register int k=i;k<j;++k){
                    f[i][j]=min(f[i][j],f[i][k]+f[k+1][j]-(int)(s[k]==s[j]));//转移
                }
            }
        }
    }
    printf("%d",f[1][n]);
    return 0;
}

#include<bits/stdc++.h>
using namespace std;

const int si_1=2e2+5;
const int si_2=4e4+10;

int head[si_1],edge[si_2],Next[si_2],tot=0;
int pos[si_1],c[si_1],dfn[si_1],low[si_1];
int Time,cnt,ins[si_1];

stack<int>q;
int n;

void add(int u,int v){
	edge[++tot]=v;
	Next[tot]=head[u];
	head[u]=tot;
}

void tarjan(int u){
	int i,v,tmp;
	dfn[u]=low[u]=++Time;
	q.push(u);
    ins[u]=1;
	for(i=head[u];i;i=Next[i]){
		v=edge[i];
		if(!dfn[v]){
			tarjan(v);
			low[u]=min(low[u],low[v]);
		}
		else if(ins[v]){
            low[u]=min(low[u],dfn[v]);
	    }
    }
	if(dfn[u]==low[u]){
		cnt++;
		do{
			tmp=q.top();
            q.pop();
            ins[tmp]=0;
			c[tmp]=cnt;
		}while(tmp!=u);
	}
}
int main(){
	int ans=0;
	scanf("%d",&n);
	for(register int i=1,v;i<=n;i++){
        while(scanf("%d",&v)&&v) add(i,v);
    }
	for(register int i=1;i<=n;i++){
        if(!dfn[i]) tarjan(i);
	}
    for(register int u=1;u<=n;u++){
		for(register int i=head[u];i;i=Next[i]){
			if(c[u]!=c[edge[i]]) pos[c[edge[i]]]++;
        }
    }
	for(register int i=1;i<=cnt;i++){
        if(!pos[i]) ++ans;
    }
	printf("%d\n",ans);
	return 0;
}

#include<bits/stdc++.h>
using namespace std;

#define int long long
namespace fastIO{
    #define spc ;putchar(' ');
    #define ent ;puts("");
    template<typename T>inline void read(T &x){
        x=0;int f=1;
        char ch=getchar();
        while(ch<'0'||ch>'9'){
            if(ch=='-')f=-1;
            ch=getchar();
        }
        while(ch>='0'&&ch<='9'){
            x=(x<<1)+(x<<3)+(ch^48),ch=getchar();
        }x*=f;
    }
    template<typename T>inline void write(T x){
        if(x<0){
            x=-x,putchar('-');
        }
        if(x>9){
            write(x/10);
        }putchar(x%10+48);
    }
    template<typename T>inline void readrange(T *a,int begin_,int end_){
        if(begin_<=end_){
            for(register int i=begin_;i<=end_;++i){
                read(a[i]);
            }return;
        }
        for(register int i=begin_;i>=end_;--i){
            read(a[i]);
        }return;
    }
}using namespace fastIO;

const int si=4e5+10;
int n,m;
int a[si];
struct segment_tree{
    int l,r;//dat interval
    int value,lazy;
    #define l(o) t[o].l
    #define r(o) t[o].r
    #define sum(o) t[o].value
    #define tag(o) t[o].lazy
    #define lson(o) o<<1
    #define rson(o) o<<1|1
}t[si*4];

inline void pushdown(int p){
    if(tag(p)){
        sum(lson(p))+=tag(p)*(r(lson(p))-l(lson(p))+1);
        sum(rson(p))+=tag(p)*(r(rson(p))-l(rson(p))+1);//update information
        tag(lson(p))+=tag(p),tag(rson(p))+=tag(p);//spread tag 
        tag(p)=0;//delete
    }return;
}

inline void build(int p,int l,int r){
    l(p)=l,r(p)=r;
    if(l==r){
        sum(p)=a[l];//or a[nr];//value of leaf
        return;
    }
    int mid=(l+r)>>1;
    build(lson(p),l,mid);
    build(rson(p),mid+1,r);
    sum(p)=sum(lson(p))+sum(rson(p));//Update(or minv,maxv,sin,cos……)
}

inline void modify(int p,int x,int val){
    int l=l(p),r=r(p);
    if(l==r){//find leaf
        sum(p)=val;return;
    }
    int mid=(l+r)>>1;
    if(x<=mid){//in leftson
        modify(lson(p),x,val);
    }
    else{//in rightson
        modify(rson(p),x,val);
    }
    sum(p)=sum(lson(p))+sum(rson(p));//Update
}

inline int query(int p,int l,int r){
	int res=0,nl=l(p),nr=r(p); 
    if(l<=nl&&nr<=r){
        return sum(p);
	}
    pushdown(p);
    int mid=(nl+nr)>>1;
    if(l<=mid){
        res+=query(lson(p),l,r);
	}
    if(r>mid){
        res+=query(rson(p),l,r);
	}
    return res;
}

inline void update(int p,int l,int r,int v){
    int nl=l(p),nr=r(p);
    if(l<=nl&&nr<=r){
        sum(p)+=v*(nr-nl+1);
        tag(p)+=v;
        return;
    }
    pushdown(p);
    int mid=(nl+nr)>>1;
    if(l<=mid){
        update(lson(p),l,r,v);
    }
    if(r>mid){
        update(rson(p),l,r,v);
    }
    sum(p)=sum(lson(p))+sum(rson(p));
}

signed main(){
    read(n),read(m);
    for(register int i=1;i<=n;++i){
        read(a[i]);
    }
    build(1,1,n);
    while(m--){
        int op,x,y,k;
        read(op),read(x),read(y);
        if(op==1){
            read(k);
            update(1,x,y,k);
        }
        else{
            write(query(1,x,y))ent;
        }
    }
    return 0;
}
