杂-对于DP的一点理解

发表于 2021-04-17

关于DP

四要素

状态（重中之重）

都说推方程式 $\text{dp}$ 里最简单的，设计出好的状态才是难的。

一般我们都是设 $f_{i,j}$ 表示什么什么东西。

然后个人觉得状态就分了两个部分：阶段和条件。

  • 阶段

这种东西都是在循环最外层枚举的（比如 $\text{Floyd}$ 的 $k$），在状态里面体现出来也就是：

前i个,第i个,对于前i段……

当然你也可以有多个“阶段”，不过要分清楚顺序。

  • 条件/决策/……

这个其实就是状态的核心了。条件类型的话一般都是看题目条件而定。

比如： 用了j的空间（背包）,疲劳度为k（某道题）,走到第j个格子……

不过还有一种我将其称为“决策型”，为什么呢？

因为这种东西就是决定“状态”里的“决策”的，

比如 0表示休息，1表示跑,0表示max，1表示min 等等。

这些一般在决策需要分情况而且需要在状态里面体现的时候有用这种状态

方程

推得时候要结合着状态来，想清楚状态从哪里来，怎么转移。

然后有些题式需要分类讨论的，所以分情况就好。

不过注意，在找情况的时候要做到不重不漏，也就是不能少情况或者多情况。

有限制的转移可以压进状态，也可以判断解决转移。

转移的时候方程一般都是：$f_{i,j}=\operatorname{fuction}(f_{i,j},f_{……})$

为了保证“无后效性”，就要做到等式左边未知而右边全部已知。

如果你觉得方程是对的，但是右边有未知，就尝试用条件把它变成已知（或者再看看你的方程是不是推错了）。

边界

初始值，根据状态来就好。

如果在外面初始化了就不用再循环里初始化，

当然如果你在循环里初始化那就没必要在外面写了（只要不漏就行）。

还有，memset的时候不要初始化double类型的东西。（其实个人觉得最好不用memset），也要注意它放在哪个位置（顺序）。

目标

也就是答案，根据你设的状态就行。

不过有的时候要在全部 $f$ 里面累加或者取最值。

也有可能循环出来了还要和原来的某种情况取最值（牛的旅行那道$\text{Floyd}$就是）。

题解 CF1504A 【Déjà Vu】

这场有那么一丢丢毒瘤。

不过A题还是一如既往的送分。

看一下题面有啥要求。

if there is no solution, output “NO”.

那么考虑怎么判无解（很明显无解情况少一点）。

这里要求你添加 'a' 之后不可以出现回文。

思考一下：只有原串全部是 'a' 才无解。

为什么？

如果你的字符串里有任何 'a' 以外的东西。

因为回文是需要像 a[i]=a[n-i+1] 一样的对应的。

可以直接把 a 放到这个不同字符对应的位置上。

所以一定有解。

然后考虑怎么构造。

上面那里已经说了，回文是要两个两个对应的。

找到第一个不是 'a' 的位置，在对应位置加一个 'a'即可。

If there are multiple solutions, you may print any.

这里多解任意输出即可，所以只要合法就行。

code:



#include<bits/stdc++.h>
using namespace std;

int T;

int main(){
	cin>>T;
	for(register int _=1;_<=T;++_){
		string s;
		cin>>s;int n=s.size();
		s=' '+s;
		int fuc=0;
		for(register int i=1;i<=n;++i){
			if(s[i]!='a'){
				fuc=0;break;
			}
			fuc=1;
		}
		if(fuc==1){
			cout<<"NO\n";
			continue;
		}//无解
		cout<<"YES\n";
		for(register int i=1;i<=n;++i){
			if(s[n-i+1]!='a'){//找到第一个位置
				for(register int j=1;j<i;++j){
					cout<<s[j];
				}cout<<"a";
				for(register int j=i;j<=n;++j){
					cout<<s[j];
				}cout<<endl;
				break;
			}
		}
	}
	return 0;
}

P1892 [BOI2003]团伙

description:

在某城市里住着n个人，任何两个认识的人不是朋友就是敌人，而且满足：

1、我朋友的朋友是我的朋友；

2、我敌人的敌人是我的朋友；

所有是朋友的人组成一个团伙。告诉你关于这n个人的m条信息，即某两个人是朋友，或者某两个人是敌人，请你编写一个程序，计算出这个城市最多可能有多少个团伙？

solution:

比较简单，关系$1$直接用并查集维护。

考虑关系$2$ ，因为并查集在维护的时候会把两个同集合元素都指向树根。

所以用另一个并查集维护敌人然后再加入会有一内内恶心（似乎是没法做）。

这里考虑维护 $i$ 这一个人的所有敌人。

那么在遇到一个新的敌人的时候，如果他之前没有敌人，那么这个敌人就是他目前的第一个敌人。直接加入到enemy里面。

反之则把这个新的敌人与他之前的enemy合并到friend里。

然后因为每一次维护都最多有两个敌人，可以不用开二维数组，直接用 $ene[i]$ 表示这个人最后一个未被维护的敌人即可。

Auoj1550 打击犯罪

description：

某个地区有n(n≤1000)个犯罪团伙，当地警方按照他们的危险程度由高到低给他们编号为1-n，他们有些团伙之间有直接联系（一条边），

但是任意两个团伙都可以通过直接或间接的方式联系，这样这里就形成了一个庞大的犯罪集团，

犯罪集团的危险程度由集团内的犯罪团伙数量唯一确定，

而与单个犯罪团伙的危险程度无关（该犯罪集团的危险程度为n）。现在当地警方希望花尽量少的时间（即打击掉尽量少的团伙），

使得庞大的犯罪集团分离成若干个较小的集团，并且他们中最大的一个的危险程度不超过n/2。

为达到最好的效果，他们将按顺序打击掉编号1到k的犯罪团伙，请编程求出k的最小值。

solution：

十分精妙的一道题。

看起来是用并查集维护。

不过如果你正常维护的话就要考虑这些情况：

对于每一个被del的点，将它 siz[rt(i)]-=1
令其的pa=自身，siz=1
对于所有和它相连的非父亲节点，pa=自身，siz=1；
然后删去该边。
对于与它相连的节点v。
没删的话都会在一个集合里，所以找到它的第一条边，Union。
反之则它消失(无用。
然后再考虑它是父亲节点的情况，这时候就必须修改整个集合。

所以不好写。

考虑倒着维护，先让他们都被打掉，然后一个一个复活。

当出现大于等于 $n/2$ 的值时统计当前情况要删的 $k$ 就行了。

Auoj 1573 构造完全图（走廊泼水节）

description:

对于完全图G，若有且仅有一棵最小生成树为T，则称完全图G是树T扩展出的。

给你一棵树T，找出T能扩展出的边权和最小的完全图G。

输入

第一行N表示树T的点数；

接下来N−1行三个整数Si,Ti,Di；描述一条边（Si,Ti）权值为Di；

保证输入数据构成一棵树。

输出

输出仅一个数，表示最小的完全图G的边权和。

N≤10^5,1≤Di≤10^5

solution:

因为给出的树是唯一最小生成树。

我们干脆在加边加边（然后……）的时候直接跑一个$\text{kruskal}$ 的“近似”算法。

考虑先把原有边排序，然后跑一个类似于mst的东西。

然后根据mst的推论。我们对于一条边 $(x,y)=val$ 考虑它们两个所在的不同集合。

然后对于另外的异于$(x,y)$ 的两个分别在$(x,y) $的集合之中的 $(u,v)$

我们考虑连另外的一条边，为了之后总值最小，我们连 $val+1$

然后这两个集合合并之前就要连 $|X|\times|Y|-1$条边，权值是 $val+1$。（X，Y是x,y所在的集合）

所以判完不连通之后才能合并。

然后这里是用生成树的性质做的。

所以我们只用考虑两个集合之间的边就好（可以想一下为什么）。

然后在过程里加一下就好。

那么代码：

//sum=原来的权值和
for(register int i=1;i<n;++i){
	int rx=root(e[i].x),ry=root(e[i].y);
	if(rx!=ry){
		sum+=(siz[rx]*siz[ry]-1)*(e[i].val+1);	
		pa[rx]=ry,siz[ry]+=siz[rx];
	}
}

发展采矿业当然首先得有矿井，小C 花了上次探险获得的千分之一的财富请人在岛上挖了n口矿井，但他似乎忘记考虑的矿井供电问题……

开始想到的是这样的一个贪心：

考虑用Kruskal先求出最小生成树。
然后对于生成树的每一个节点。
考虑和它相邻的边。
如果所有和它相连的边都比它的值大。
那么建立电站。
否则连边。
但是png里构造的数据可以卡掉这个不成熟的贪心

如你所见，这个贪心被卡爆了

如果最小生成树是这样子的：

(5)–6–(7)–10–(1)

那么就死透了（可以自己模拟一下，按照那个贪心出来是13，但是有一个12的答案）。

那么这里有一个神奇的做法。

因为我们的贪心被卡掉就是因为在点上的元素不好搞。

都说这是图论图论，mst，重点就是在边的处理上。

干脆我们就把这个点值化成边值。

也就是在原图上新加一个点（超级源点）

它和 $1-n$ 的节点全部相连。

边权是在 $i$ 建立电站的cost $v_i$

也就相当于我们把每一个点化成边来处理.

然后在原图上跑一遍最小生成树就好了。

【国家集训队2012】[Tree I]

description:

描述

原题来自：2012 年国家集训队互测
给你一个无向带权连通图，每条边是黑色或白色。让你求一棵最小权的恰好有 need 条白色边的生成树。题目保证有解。

输入
第一行 V,E,need 分别表示点数，边数和需要的白色边数。
接下来 E 行，每行 s,t,c,col 表示这边的端点（点从 0 开始标号），边权，颜色（0 白色，1 黑色）。

输出
一行表示所求生成树的边权和。

solution:

考虑我们直接给原图跑一遍最小生成树。

无非就是三种情况：

  1. 刚好有$need$条白边
  2. 比$need$少了一些
  3. 比$need$多了一些

对于1我们不用干啥。

然后考虑，怎么样在求最小生成树的同时增减白边的条数呢？

这里有一种神奇的想法。

我们给所有白边都加上一个值$val$

那么有可能加到某个值，一条白边就被加入（踢出）mst了。

怎么确定这个值？

简单思考一下发现 $val$ 是满足单调性的。

所以考虑二分答案。

如果求出来的真实白边数 $real$ 大于$need$ ，考虑就要增加 $val$ 来减少白边。

反之亦然。

然后在$need \leq real$ 的时候更新答案就行了（因为这个时候才是可行的，可以自己证明一下。

这里给出代码：


#include<bits/stdc++.h>
using namespace std;

const int si=2e5+10;
int n,m,need;
int pa[si];

struct node{
	int x,y,val,ty;
	bool operator < (const node &u)const{
		if(val==u.val) return ty<u.ty;
		return val<u.val;
	}
}e[si];
int root(int x){
	if(pa[x]!=x){
		pa[x]=root(pa[x]);
	}
	return pa[x];
}
bool Union(int x,int y){
	int rx=root(x),ry=root(y);
	if(rx==ry){
		return false;
	}
	pa[rx]=ry;
	return true;
}

void kruskal(int &sum,int &real){
	sort(e+1,e+1+m);
	for(register int i=1;i<=n;++i){
		pa[i]=i;
	}
	for(register int i=1;i<=m;++i){
		if(Union(e[i].x,e[i].y)){
			sum+=e[i].val;
			real+=(int)(e[i].ty==0);
		}
	}
	return;
}

int main(){
	cin>>n>>m>>need;
	for(register int i=1;i<=m;++i){
		cin>>e[i].x>>e[i].y>>e[i].val>>e[i].ty;
		++e[i].x,++e[i].y;
	}
	int l=-114,r=114;
	int res=0;
	while(l<=r){
		int mid=(l+r)>>1;
		for(register int i=1;i<=m;++i){
			if(e[i].ty==0) e[i].val+=mid;
		}
		int sum=0,real=0;
		kruskal(sum,real);
		if(real>=need) res=sum-(need*mid),l=mid+1;
		else{
			r=mid-1;
		}
		for(register int i=1;i<=m;++i){
			if(e[i].ty==0) e[i].val-=mid;
		}
	}
	printf("%d\n",res);
	return 0;
}

然后对于每一个牧区求和它联通的点最远的距离 D[i]

然后枚举所有不连通的点连边之后的情况然后取最值。

值得注意的是连了之后可能还不如原来的大，所以要在最后再取一次最值。

#include<bits/stdc++.h>
using namespace std;

const int inf=114514333;
const int si=500;
int n;
double f[si][si];
struct node{
	int x,y;
}p[si];
double clac(int u,int v){
	return (double)sqrt((p[u].x-p[v].x)*(p[u].x-p[v].x)+(p[u].y-p[v].y)*(p[u].y-p[v].y));
}

void Floyd(){
	for(register int k=1;k<=n;++k){
		for(register int i=1;i<=n;++i){
			for(register int j=1;j<=n;++j){
				f[i][j]=min(f[i][j],f[i][k]+f[k][j]);
			}
		}
	}
}

double D[si];//直径（可以看做对于每个点的） 
double D1,D2=inf;
int main(){
	cin>>n;
	for(register int i=1;i<=n;++i){
		cin>>p[i].x>>p[i].y;
	}
	for(register int i=1;i<=n;++i){
		for(register int j=1,o;j<=n;++j){
			scanf("%1d",&o);
			if(o==1) f[i][j]=clac(i,j);
			else if(i!=j) f[i][j]=(double)inf;
		}
	}
	Floyd();
	for(register int i=1;i<=n;++i){
		for(register int j=1;j<=n;++j){
			if(f[i][j]!=inf){
				D[i]=max(D[i],f[i][j]);
			}
			D1=max(D1,D[i]);
		}
	}
	for(register int i=1;i<=n;++i){
		for(register int j=1;j<=n;++j){
			if(f[i][j]==inf){
				double w=clac(i,j);
				D2=min(D2,D[i]+w+D[j]);
			}
		}
	}
	double ans=max(D1,D2); 
	printf("%.6lf\n",ans);
	return 0;
}

			}

	}

solution

这个题是一个比较典型的 小区间推大区间。

因为你会发现老张走路不一定是一条线。

也就是说正常的 线性$\text{dp}$ 是没有办法做这种题的（不能按从前往后或者从后往前推）。

那么这个时候就要联想到 区间$\text{dp}$ 这个“不按线性顺序”更新的东西

注意到这里老张是想“关灯”且尽量早关 , 也就是说他如果经过了一个灯他必定会关掉，不然再折回来就会浪费，不是最优的。

也就是说，他走路的顺序一定是类似于这种的（直线也包括在内）：

那么也就是说，对于一个区间,老张如果站在这个区间的端点的时候，那么他必定早已关完了这个区间所有的灯。

所以我们就可以从 $f[start][start]$ 这个最小的区间慢慢推到最大的 $f[1][n]$ 这个区间。

然后一个区间是有两个端点的，所以要用 $[0/1]$ 区分一下（因为不同的端点可能有不同的值，也就是方向的不同导致了结果的不同）。

至于怎么转移呢？

我们这里完全没有必要用一个 $k$ 去枚举上一个决策点了，因为我们推出来了这一条：

他如果经过了一个灯,那么他必定会关掉。

所以，比如在区间左端点的时候，那么就可以直接从 $l+1$ 这个位置转移过来。

所以每一次推大区间的时候都是 每次$+1$ 这样子推过去的。

当然你转移的时候也要考虑这个小区间是在左端点过来还是右端点过来。

（你也看到了，上图的路径是有重复的，所以不一定是离他最近的端点转移过来的）

然后考虑怎么计算花费，这里直接计算一下小区间的功耗然后加上扩展出来的（也就是端点的花费）就好了。

#include<bits/stdc++.h>
using namespace std;

const int si=60;
int a[si],b[si],sum[si];
int n,m,root;
int f[si][si][2];

int main(){
	scanf("%d%d",&n,&root);
	memset(f,127,sizeof(f));
	for(register int i=1; i<=n;i++){
		scanf("%d%d",&a[i],&b[i]);
		sum[i]=sum[i-1]+b[i];
	}
	f[root][root][0]=f[root][root][1]=0;
	for(register int l=2;l<=n;l++){
		for(register int i=1;i+l-1<=n;i++){
			register int j=i+l-1;
			f[i][j][0]=min(f[i+1][j][0]+(a[i+1]-a[i])*(sum[i]+sum[n]-sum[j]),f[i+1][j][1]+(a[j]-a[i])*(sum[i]+sum[n]-sum[j]));
			f[i][j][1]=min(f[i][j-1][0]+(a[j]-a[i])*(sum[i-1]+sum[n]-sum[j-1]),f[i][j-1][1]+(a[j]-a[j-1])*(sum[i-1]+sum[n]-sum[j-1]));
		}
	}
	int ans=min(f[1][n][0],f[1][n][1]);
	printf("%d",ans);
	return 0;
}

题解 CF1132F 【Clear the String】

区间Dp

看到这种可以区间区间删的东西。

直接就可以想到区间DP。

那么设 $f[i][j]$ 表示把 $[i,j) $ 删完 的最小代价。

考虑如何转移。

记得有一道 CQOI 的题，好像是叫涂色来着，和这道题的做法比较像。

那道题的经典思路就是判断 $s[j]=s[k]$ 是否为真，为真即表示这个区间 $[j,k)$ 可以涂上一种颜色。

看到这一道题，我联想到的就是雷同于涂色那道题的做法。

那么还是：判断 $s[j]=s[k]$ 是否为真。

不过这一道题是说直接删去，我们就可以想到，如果上式为真，那么合并这个区间的价值就直接是 $f[i][k]+f[k+1][j]$

反之，我们需要多删去 $s[j]$ 和 $s[k]$ 中的任意一个，那么就要花费 1 的代价。

所以不难搞出方程： f[i][j]=min(f[i][j],f[i][k]+f[k+1][j]-(int)(s[k]==s[j]));

还有，边界条件。

我们根据状态的定义，直接把所有的 $f[i][i]$ 都赋值为 1。

当然，如果我们在外面初始化了，在转移的时候，如果 $i=j$，我们直接 continue 掉就可以了。

代码：


#include<bits/stdc++.h>
using namespace std;

const int si=1003;
int n;
char s[si];
int f[si][si];

int main(){
    scanf("%d",&n);
    cin>>s+1;//把字符串长度加一
    memset(f,0x3f,sizeof(f));//先赋一个初值
    for(register int i=1;i<=n;++i){
        f[i][i]=1;
    }
    for(register int len=2;len<=n;++len){
        for(register int i=1;i+len-1<=n;++i){
            register int j=i+len-1;
            if(i==j){
                continue;
            }//i=j直接跳过
            else{
                for(register int k=i;k<j;++k){
                    f[i][j]=min(f[i][j],f[i][k]+f[k+1][j]-(int)(s[k]==s[j]));//转移
                }
            }
        }
    }
    printf("%d",f[1][n]);
    return 0;
}

const int si=4e5+10;
int n,m;
int a[si];
struct segment_tree{
    int l,r;//dat interval
    int value,lazy;
    #define l(o) t[o].l
    #define r(o) t[o].r
    #define sum(o) t[o].value
    #define tag(o) t[o].lazy
    #define lson(o) o<<1
    #define rson(o) o<<1|1
}t[si*4];

inline void pushdown(int p){
    if(tag(p)){
        sum(lson(p))+=tag(p)*(r(lson(p))-l(lson(p))+1);
        sum(rson(p))+=tag(p)*(r(rson(p))-l(rson(p))+1);//update information
        tag(lson(p))+=tag(p),tag(rson(p))+=tag(p);//spread tag 
        tag(p)=0;//delete
    }return;
}

inline void build(int p,int l,int r){
    l(p)=l,r(p)=r;
    if(l==r){
        sum(p)=a[l];//or a[nr];//value of leaf
        return;
    }
    int mid=(l+r)>>1;
    build(lson(p),l,mid);
    build(rson(p),mid+1,r);
    sum(p)=sum(lson(p))+sum(rson(p));//Update(or minv,maxv,sin,cos……)
}

inline void modify(int p,int x,int val){
    int l=l(p),r=r(p);
    if(l==r){//find leaf
        sum(p)=val;return;
    }
    int mid=(l+r)>>1;
    if(x<=mid){//in leftson
        modify(lson(p),x,val);
    }
    else{//in rightson
        modify(rson(p),x,val);
    }
    sum(p)=sum(lson(p))+sum(rson(p));//Update
}

inline int query(int p,int l,int r){
	int res=0,nl=l(p),nr=r(p); 
    if(l<=nl&&nr<=r){
        return sum(p);
	}
    pushdown(p);
    int mid=(nl+nr)>>1;
    if(l<=mid){
        res+=query(lson(p),l,r);
	}
    if(r>mid){
        res+=query(rson(p),l,r);
	}
    return res;
}

inline void update(int p,int l,int r,int v){
    int nl=l(p),nr=r(p);
    if(l<=nl&&nr<=r){
        sum(p)+=v*(nr-nl+1);
        tag(p)+=v;
        return;
    }
    pushdown(p);
    int mid=(nl+nr)>>1;
    if(l<=mid){
        update(lson(p),l,r,v);
    }
    if(r>mid){
        update(rson(p),l,r,v);
    }
    sum(p)=sum(lson(p))+sum(rson(p));
}

最小生成树

定义：

简单来说，最小生成树是一张带权无向图里面由全部 $n$ 个顶点，和边集中的 $n-1$ 条边构成的原图的一颗生成树，且所有边权之和是所有生成树中最小的（可能有多个情况）。

而且它一定包含原图中最小的边。

推论：

设一张无向图 $G=(V,E)$ ，从 $E$ 中选出 $k<|V|-1$条边构成 $G$ 的一个生成森林，然后再从剩余的 $|E|-k$ 条边中选出 $|V|-1-k$ 条边加入森林中，让它成为 $G$ 的生成树，并且选出的$\sum w$最小。

那么，这个生成树一定包含 $|E|-k$ 条边里面连接生成森林的两个不连通节点的权值最小的边。

很拗口对吧？

不过请好好理解，因为这个推论就是接下来的 $\text{Kruskal}$ 的基础了。

Kruskal

这是一种求最小生成树的高效算法。

它维护图的最小生成森林。

开始的时候生成森林是空的，每一个节点就是一颗独立的树。

然后我们用上述推论维护森林就好了。

首先先搞一个并查集出来。对于每一个点初始化。

接下来按照边权升序排个序。

然后扫一遍每个边。

如果这一条边所连得两条边$(u,v)$已经联通了。那么继续。

but，如果没有联通，根据这一条：

这个生成树一定包含 $|E|-k$ 条边里面连接生成森林的两个不连通节点的权值最小的边。

所以满足了条件，那么我们就把这一条边加入到最小生成树里。

顺便合并一下 $(u,v)$ （相当于让它联通）

代码：

#include<bits/stdc++.h>
using namespace std;

const int si=2e6+2;

int n,m;
int ans;
int pa[si];

struct edge{
    int x,y,val;
    bool operator < (const edge &u)const{
        return val<u.val;
    }
}ed[si];

int root(int x){
    if(pa[x]!=x) pa[x]=root(pa[x]);
    return pa[x];
}

bool Union(int x,int y){
    int rx=root(x),ry=root(y);
    if(rx==ry){
        return 0;
    }
    pa[rx]=ry;
    return 1;
}

int main(){
    cin>>n>>m;
    for(register int i=1;i<=m;++i){
        cin>>ed[i].x>>ed[i].y>>ed[i].val;
    }
    sort(ed+1,ed+m+1);
    for(register int i=1;i<=n;++i){
        pa[i]=i;
    }
    for(register int i=1;i<=m;++i){
        if(Union(ed[i].x,ed[i].y)) ans+=ed[i].val;
    }
    cout<<ans<<endl;
    return 0;
}

Prim

这个算法实际上不是很常用。

因为在没有堆优化的情况下，$\text{Prim}$ 的复杂度是 $\text{O} (n^2)$
的。

而且写了堆优化以后会比 $\text{Kruskal}$ 麻烦的多（某次十一集训的时候xzq就是想写 $\text{Prim}$ ，结果我用 $\text{Kruskal}$ A了几道题了他还没调好/xyx）

那么大概说一下思路。

现在已经知道 $\text{Kruskal}$ 是维护一整个生成森林。

其实 $\text{Prim}$ 就和它稍微有点不同而已，它只是维护了最小生成树的一部分。而且它最开始确定 $1$ 号节点属于最小生成树。

它每一次找到一条权值最小的，且满足它连接的其中一个点 $u$ 已经被选入最小生成树里，另一个点 $v$ 则未被选中的边。

具体实现是这样子的：

维护一个数组 $dis[\ ]$ ,如果 $u$ 没有被选入，那么 $dis[u]$ 就等于 $u$ 和已经被选中点之间的连边中权值最小的边的权值。

反之 $dis[u]$ 就等于 $u$ 被选中的时候选出来那条权值最小边的权值。

然后怎么判是否选中呢？

维护一个$vis[]$ 即可。然后从 $vis[]=\text{false}$的节点中 （也就是未被选中的节点） 选出一个 $dis[]$ 最小的标记它

然后扫描和这个被选点的所有直接连它的边，更新另外一个端点的 $dis$ 。

最后生成树的权值和就是 $\sum^{n}_{i=2} dis[i]$。

代码：



#include<bits/stdc++.h>
using namespace std;

const int si=5e3+10;
int n,m;
int a[si][si];
int dis[si];
bool vis[si];

int main(){
	cin>>n>>m;
	memset(a,0x3f,sizeof a);
	for(register int i=1;i<=n;++i){
		a[i][i]=0;
	}
	for(register int i=1,u,v,w;i<=m;++i){
		cin>>u>>v>>w;
		a[v][u]=a[u][v]=min(a[u][v],w);
	}
	//Prim
	memset(dis,0x3f,sizeof dis);
	memset(vis,false,sizeof vis);
	dis[1]=0;
	for(register int i=1,u;i<n;++i){
		u=0;
		for(register int j=1;j<=n;++j){
			if(!vis[j]&&(u==0||dis[j]<dis[u])) u=j;
		}
		vis[u]=1;
		for(register int v=1;v<=n;++v){
			if(!vis[v]) dis[v]=min(dis[v],a[u][v]);
		}
	}
	//answer
	int ans=0;
	for(register int i=2;i<=n;++i){
		ans+=dis[i];
	}
	cout<<ans<<endl;
	return 0;
}

$O(n^2)$ DP方法

这一道题的重点在于离散化，网上有九成的题解都讲的不清晰，所以我在这里细细的讲一讲。

思路：

既然这是线性 $DP$ 。

从以下两个角度出发：

  • 我从哪里来？（当前状态可以由哪些状态得到？）

  • 我到哪里去？（当前状态可以去到哪几个状态？）

我们不难想到：

设 $f[i][j]$ 表示 $a[i]$ 变成 $re[j]$ 所需的最小操作次数。

路人：诶诶诶，你 $a[i]$ 和 $re[j]$ 哪里来的啊？

这台子拆的好！

我们设原序列为 $a[]$ ，我们想要达到的“理想”序列为 $re[]$。

那么，代码如下：

   for(register long long i=1;i<=n;++i){
	cin>>a[i];
	a[i]-=i;
	re[i]=a[i];
}
sort(re+1,re+n+1);

接下来我就讲讲为什么要这样做：

为什么是 $re[i]=a[i]-i$

很好理解，我在这里举个例子。

（这里举的例子是POJ3666，因为离散的方法和此题一样，所以我就以POJ3666作为例子讲解这题的离散化）

先不管这道题的要求。我们先假设加一减一的操作变为了修改操作（POJ3666）。

那么：

$n=5$

$a[]:{1,3,2,3,5}$

如果我们想要修改它，使得它是单调不减（这里也是POJ3666）的。要怎么修改？

很明显，我们的第一思路是这么修改：

$a[]:{1,2,3,4,5}$

很明显，因为这里有重复的 $3$ 。所以，我们需要更多的次数去修改它。

为了避免这种情况，我们给每个 $a[i]$ 减去 $i$。

为什么呢？

因为 $i$ 在这里一定是严格递增的（你从头往后扫，下标难道不递增？），现在看一下减去 $i$ 之后是怎么样的：

$re[]:{ 0,1,-1,-1,0}$

现在还是有重复的，不过先不要着急。

我画一个图。

这是你的 $re$ 序列（不下降）：

这是你的 $i$ （所有的 $i$ 组成的序列）

那我们将这两个序列合并，这个合并的序列一定是这样的：

其实就是一个不等式的加减。

举个例子：

$re[]={ 1,2,3,3,3}$

$i[]={1,2,3,4,5}$

合并之后：

$2,4,6,7,8$,是不是满足严格递增？

好的，这里可能会有一点绕（看完就能理解的）。

我们再重新看一看:

$re[]:{ 0,1,-1,-1,0}$

现在我们只需要使这一个 $re$ 中不下降的元素尽量多，那么我们需要修改的次数就会尽量的少（最优）（也就是要求一个最长不下降子序列再减去元素个数）。

好了，把这一个离散的思路放到我们这一题，只需要改一下条件就可以啦。

现在离散化已经搞定了。那么是时候看方程了。

方程（这里推得过程省去，都做到这种题了，这种普通的线性 $DP$ 方程不可能推不出来）：

$$ \begin{cases}minn=\min(minn,f[i-1][j])\f[i][j]=minn+|a[i]-re[j]|\end{cases}$$

（最开始的时候 $minn$ 要赋值为一个超级大的值）

#include<bits/stdc++.h> 
using namespace std;

long long f[3001][3001];//f[i][j]表示a[i]变为re[j]时的最小操作次数 
long long n;
long long a[3001];//原序列 
long long re[3001];//目标序列 
long long minn,res=2e18;//这里要赋很大的值，否则以CF的毒瘤数据你会WA。

long long abs_ll(long long a){
	return a>=0?a:-a;
}

namespace Accepted{
	void zjk_ak_ioi(){
		cout<<res<<endl;
		exit(0);
	}//对神zjk的膜拜 （这里不用管啦，你直接输出res就可以了）
} 

int main(){
	cin>>n;
	for(register long long i=1;i<=n;++i){
		cin>>a[i];
		a[i]-=i;
		re[i]=a[i];//离散化 
	}
	sort(re+1,re+n+1);//排一遍，使目标序列严格递增 
	for(register long long i=1;i<=n;++i){
		f[1][i]=abs_ll(a[1]-re[i]);//给f一个初值，这里是预处理
	}
	for(register long long i=2;i<=n;++i){
		minn=2e18;//这里每一次都要赋值，否则会错（因为每次操作都要用 ） 
		for(register long long j=1;j<=n;++j){
			minn=min(minn,f[i-1][j]);
			f[i][j]=minn+abs_ll(a[i]-re[j]);//转移方程 
		}
	}
	for(register int i=1;i<=n;++i){
		res=min(res,f[n][i]);//从整个序列的情况去找最小值 
	}

}
