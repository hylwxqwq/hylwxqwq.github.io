P6018 [Ynoi2010]Fusion tree

sol： BY：FZzzz

首先这种题其实是个很经典的 trick 就是对于每个点只维护它的所有儿子。然后对于父亲单独处理。对于父亲的处理是平凡的，不再赘述。

然后考虑一个节点，我们要处理的就是单点修改和全局加，然后要查询全局异或和。

不对啊你发现每次全局加的都是一啊，这个肯定得有用啊。

考虑加上一到底会对一个数造成什么样的影响。

不就是把最低位的零变成一然后把它下面的所有一都变成零吗，这个怎么处理啊。

我们从低位到高位建一个 trie，然后按位考虑每一位，对于零就直接把它变成一，然后对于一就变成零，但是还要递归下去。

所以直接就交换一下儿子然后递归下去修改，维护一下子树异或和就行了。

CF414B

des：

如果一个数列中，后一个数都能被前面一个数整除，那么就叫这个数列为好数列。输入n，k，求数列中最大元素为n，数列长度为k的好数列的种数。

sol：比较简单。我们根据题目的要求搞就可以了。

不过在设计状态的时候就要把枚举长度的放在外面了。

然后再开一维枚举k，就是乘上去的那个数。

然后根据状态定义有这么一个方程：$f[i][j]=(f[i][j]+f[i-1][k*j])$;

记得取模。

P4551

des：

给定一棵n个点的带权树，结点下标从1开始到N。寻找树中找两个结点，求最长的异或路径。

异或路径指的是指两个结点之间唯一路径上的所有边权的异或。

sol：根据我们异或的性质（异或两次就是它本身），有一个想法:

先DFS求出root到每一个点的dist。

然后$dist[x]\ xor\ dist[y]$ 就是 $x -> y$ 的异或路径长。

用一颗01Trie来维护就行了。

des:

一条高铁线路G39依次经过n个火车站，编号1~n,现在有未来K天的m条的订票信息，请问每次安排至少多少座位可满足订票信息。

信息形式为“i j k”,表示第i天出行，从j上车，从k下车。座位可同时使用，但两人不可同时占用同一座位。（比如第一位乘客在3号站下车，同时第二位乘客在3号火车站上车，这样只需要一个座位即可） 。

AUOJ1365 C老师的母鸡

des:

给定 $n,t$ ，在长度为 $n$ 的序列中找到长度最小的一个子段，使得字段和大于等于 $t$。 （均为正整数）。

sol:: 尺取法模板。我们让 $l$ 从 $[1,n]$ 之间循环。从 $l$ 开始往后枚举 $r$ （每次往后加一） 如果当前长度的满足要求（往后就不会更优了），记录当前的长度，和原来的最优解比较。最后输出最优解就可以了。

P1824 cow’s new house

des::

Farmer John建造了一个有N(2<=N<=100,000)个隔间的牛棚，这些隔间分布在一条直线上，坐标是x1,…,xN (0<=xi<=1,000,000,000)。

他的C(2<=C<=N)头牛不满于隔间的位置分布，它们为牛棚里其他的牛的存在而愤怒。为了防止牛之间的互相打斗，Farmer John想把这些牛安置在指定的隔间，所有牛中相邻两头的最近距离越大越好。那么，这个最大的最近距离是多少呢？

sol:看到 这个最大的最近距离 这种东西，应该能直接先想到二分答案。

定义变量ans，储存当前优解。定义闭区间[left, right]，代表程序当前正在此闭区间内寻找答案（寻找潜在的比ans更优的解）（与前两种方法不同的是，最优解不一定要属于该闭区间）。

令 $mid = (left + right)/2$

若$mid$为解，则$ans = max(ans, mid), left = mid + 1.$此时我们更新了最优解，同时在最优解的右侧寻找潜在的更有解。

若$mid$不为解，则$right = mid - 1$.$mid$不是解，因此我们在$mid$左边寻找更优解。

重复上述过程，直到$left > right$时跳出循环，$ans$即为最优解.

注意，当$left == right$时，也必须要在此区间内进行判断，因为当前还不能确定该区间内是否存在更优解。

AUOJ1367

des ：

给定长度为 $n$ 的序列。将其分为 $k$ 段，使得每一段的最大值-最小值 最小。

sol ：二分答案，主要是这个题面是我简化了的。

所以很明显可以想到二分答案。

其实二分答案的核心就是那个 $check()$ 。

我们每一次二分答案的值是 $mid$ .

那么对于 $check$ 函数所传的参数，我们只需要结合题目条件判断它是否可行就好（这里一般是 $\text{O}(n)$ 的。

那么这一道题就有几种情况。

  1. 当前枚举到的 $maxvalue-minvalue$ 大于 $mid$ 那么要重新分一次段。

  2. 当前所枚举出的总段数太大了。不可行。

  3. 枚举完之后 满足要求，当前解可行。

不过要稍微注意一下细节。

这里给出check的实现： 

bool valid(int value){
	int tot=0;//当前分了多少段 
	int maxv=-inf,minv=inf; 
	for(register int i=1;i<=n;++i){
		maxv=max(maxv,a[i]),minv=min(minv,a[i]);
		if(maxv-minv>value){
			++tot,maxv=minv=a[i];
		}//update
		if(tot>k){
			return false;
		}
	}
	++tot;//这里一定要记得加上！
	return tot<=k;
}

AUOJ 1375 Graph

des:

在2117年c国为了缓减交通压力开发了空中汽车，为了保证夜间行驶的安全在各个路段安装了一系列路
灯进行照明。

已知路灯的照明范围是一个以路灯所在的点为顶点的道路最低的直线为底边的等腰直角三
角形。

汽车的行驶区域必须有路灯的照明。我们为了简化问题将道路简化为一条直线。建立平面直角坐
标系，并定义道路的最低点所在的直线为X轴。

给定所有路灯在直角坐标系的位置以及一个车辆需要从点
a行驶到点b

问这辆汽车所能行驶的最高高度。

输入

第一行一个整数n表示路灯的数量

接下来n行每行两个整数(x,y)表示第i个路灯所在的位置

第n+2行每行两个整数x1,x2，表示a,b两点的x坐标位置

输入数据保证n<=1e5所有的坐标均为正整数且小于等于1e4
输出
一个数表示能够行驶的最高高度
精确到小数点后3位

sol ：

题面有一点长，大意是给定一堆直角三角形的坐标。求一条函数
$y=mid$ 使得它在 $[a,b]$ 之间的每一个点都有三角形覆盖。

求 $mid_{\text{max}}$ 。

考虑二分答案。因为题目要求的是那一条常数函数的解析式，直接二分它的那个$y$ 值。

为了避免实数二分。我们直接将所有读入的坐标倍增一下。

一倍就够了。（等腰RT三角形）

然后在check里面，我们维护一个差分数组。

看图：

比如说我们要维护 $B$ ，那么很明显在我们二分的值为 2 的时候，我们要维护一下差分数组的 $[2,6]$ 。

现在推广到一般情况。

若 $B(x,y)$ ，当前二分的值为 $value$ 。

那么我们做一条垂线 $BF$ （很明显$\tan \angle ABF =1$）

因为 $BF=y_B-value$

所以我们要维护的区间左端点 : $x-y+value$

右端点 : $x+y-value$

还有，因为二分答案的check总是有很多细节。所以每一次写的时候都要考虑全。

这里我们维护一次差分数组的大前提一定是 $y_B >= value$

（要不然您怎么算）

然后在看它的 $[a,b]$ （给出来的）的时候。我们还要看它和我们所有的 右端点 $x+y-value$ 的大小。如果不满足，那根本就没法覆盖。

然后我们 $\text{O}(n)$ 还原一下数组。

从 $[a,b]$ 扫一下看一下有没有 $0$ 即可（其实这个差分数组的本质就是 $i,value$ 这个点被几个三角形覆盖了，0就是没有）（另外这是要多次二分的，记得memset 0）。

记得二分开始的时候 $r$ 是所有三角形顶点的最大值，这样二分次数会少一些。

还有，输出的时候强制转类型，一定要先除2再变！！（gua了好几次）。

des:

输入

bool valid(int x){
	for(register int i=0;i<g[x].size();++i){
		if(a[x]==a[g[x][i]]){
			return false;
		}
	}

}

void dfs(int step){
	if(step>n){
		++tot;
		return ;

AUOJ 1423

C老师在玩植物大战僵尸时，可以控制僵尸进攻植物；

僵尸有m种，每种僵尸无限多，c老师可任意选择，使用第i种僵尸需要花费wi的资源，可以得到pi的攻击效果，多个僵尸的总攻击效果为各攻击效果的代数和；地图有n行，第i行最左端有若干植物，这些植物至少受到qi的攻击才能被全部消灭，然后这一行被视为攻破；c老师希望你给出花费最小的方案

输入

第一行两个非负整数，m,n;

第二行m个正整数，wi；

第三行m个正整数，pi；

第四行n个非负整数 qi；

输出

输出攻破所有行的最小值

sol ：

设 $f[i]$ 表示凑出 i 的效果的最小花费。

那么对于n个询问，我们把 $\Sigma^{i=1}_{f[q[i]]}$ 加起来就好了。

然后就是个普通背包了……
