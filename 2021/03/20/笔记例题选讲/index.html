例题选解

并查集&最小生成树

P1892 [BOI2003]团伙

description:

在某城市里住着n个人，任何两个认识的人不是朋友就是敌人，而且满足：

1、我朋友的朋友是我的朋友；

2、我敌人的敌人是我的朋友；

所有是朋友的人组成一个团伙。告诉你关于这n个人的m条信息，即某两个人是朋友，或者某两个人是敌人，请你编写一个程序，计算出这个城市最多可能有多少个团伙？

solution:

比较简单，关系$1$直接用并查集维护。

考虑关系$2$ ，因为并查集在维护的时候会把两个同集合元素都指向树根。

所以用另一个并查集维护敌人然后再加入会有一内内恶心（似乎是没法做）。

这里考虑维护 $i$ 这一个人的所有敌人。

那么在遇到一个新的敌人的时候，如果他之前没有敌人，那么这个敌人就是他目前的第一个敌人。直接加入到enemy里面。

反之则把这个新的敌人与他之前的enemy合并到friend里。

然后因为每一次维护都最多有两个敌人，可以不用开二维数组，直接用 $ene[i]$ 表示这个人最后一个未被维护的敌人即可。

Auoj1550 打击犯罪

description：

某个地区有n(n≤1000)个犯罪团伙，当地警方按照他们的危险程度由高到低给他们编号为1-n，他们有些团伙之间有直接联系（一条边），

但是任意两个团伙都可以通过直接或间接的方式联系，这样这里就形成了一个庞大的犯罪集团，

犯罪集团的危险程度由集团内的犯罪团伙数量唯一确定，

而与单个犯罪团伙的危险程度无关（该犯罪集团的危险程度为n）。现在当地警方希望花尽量少的时间（即打击掉尽量少的团伙），

使得庞大的犯罪集团分离成若干个较小的集团，并且他们中最大的一个的危险程度不超过n/2。

为达到最好的效果，他们将按顺序打击掉编号1到k的犯罪团伙，请编程求出k的最小值。

solution：

十分精妙的一道题。

看起来是用并查集维护。

不过如果你正常维护的话就要考虑这些情况：

对于每一个被del的点，将它 siz[rt(i)]-=1
令其的pa=自身，siz=1
对于所有和它相连的非父亲节点，pa=自身，siz=1；
然后删去该边。
对于与它相连的节点v。
没删的话都会在一个集合里，所以找到它的第一条边，Union。
反之则它消失(无用。
然后再考虑它是父亲节点的情况，这时候就必须修改整个集合。

所以不好写。

考虑倒着维护，先让他们都被打掉，然后一个一个复活。

当出现大于等于 $n/2$ 的值时统计当前情况要删的 $k$ 就行了。

Auoj1566 Internet

description：

某个局域网内有n(n≤100)台计算机，由于搭建局域网时工作人员的疏忽，现在局域网内的连接形成了回路，
我们知道如果局域网形成回路那么数据将不停的在回路内传输，造成网络卡的现象。因为连接计算机的网线本身不同，所以有一些连线不是很畅通，我们用f(i,j)表示i,j之间连接的畅通程度(f(i,j)≤1000)，
f(i,j)值越小表示i,j之间连接越通畅，f(i,j)为0表示i,j之间无网线连接。
现在我们需要解决回路问题，我们将除去一些连线，使得网络中没有回路，
并且被除去网线的Σf(i,j)最大，请求出这个最大值。

solution：

考虑没有回路怎么做，实际上就是搞一颗生成树。

然后让你求删边的max，其实就是总边权-mst权值就好了。

Auoj 1568 [Tyvj]

description：

Tyvj已经一岁了，网站也由最初的几个用户增加到了上万个用户，随着Tyvj网站的逐步壮大，管理员的数目也越来越多，现在你身为Tyvj管理层的联络员，希望你找到一些通信渠道，使得管理员两两都可以联络（直接或者是间接都可以）。
Tyvj是一个公益性的网站，没有过多的利润，所以你要尽可能的使费用少才可以。
目前你已经知道，Tyvj的通信渠道分为两大类，一类是必选通信渠道，无论价格多少，你都需要把所有的都选择上；还有一类是选择性的通信渠道，你可以从中挑选一些作为最终管理员联络的通信渠道。
数据保证给出的通行渠道可以让所有的管理员联通。

solution：

考虑先把必须的直接连上。

然后再剩下的里面跑一个（不完整的）最小生成树。

只需要排序的时候把必须的仍在最前面然后再按边权排序。

否则就会出现必须连的边连成了另外一条，就会挂。

Auoj 1573 构造完全图（走廊泼水节）

description:

对于完全图G，若有且仅有一棵最小生成树为T，则称完全图G是树T扩展出的。

给你一棵树T，找出T能扩展出的边权和最小的完全图G。

输入

第一行N表示树T的点数；

接下来N−1行三个整数Si,Ti,Di；描述一条边（Si,Ti）权值为Di；

保证输入数据构成一棵树。

输出

输出仅一个数，表示最小的完全图G的边权和。

N≤10^5,1≤Di≤10^5

solution:

因为给出的树是唯一最小生成树。

我们干脆在加边加边（然后……）的时候直接跑一个$\text{kruskal}$ 的“近似”算法。

考虑先把原有边排序，然后跑一个类似于mst的东西。

然后根据mst的推论。我们对于一条边 $(x,y)=val$ 考虑它们两个所在的不同集合。

然后对于另外的异于$(x,y)$ 的两个分别在$(x,y) $的集合之中的 $(u,v)$

我们考虑连另外的一条边，为了之后总值最小，我们连 $val+1$

然后这两个集合合并之前就要连 $|X|\times|Y|-1$条边，权值是 $val+1$。（X，Y是x,y所在的集合）

所以判完不连通之后才能合并。

然后这里是用生成树的性质做的。

所以我们只用考虑两个集合之间的边就好（可以想一下为什么）。

然后在过程里加一下就好。

那么代码：

//sum=原来的权值和
for(register int i=1;i<n;++i){
	int rx=root(e[i].x),ry=root(e[i].y);
	if(rx!=ry){
		sum+=(siz[rx]*siz[ry]-1)*(e[i].val+1);	
		pa[rx]=ry,siz[ry]+=siz[rx];
	}
}

Auoj 1572 New_Start (star)

description:

发展采矿业当然首先得有矿井，小C 花了上次探险获得的千分之一的财富请人在岛上挖了n口矿井，但他似乎忘记考虑的矿井供电问题……

为了保证电力的供应，小C 想到了两种办法：

在这一口矿井上建立一个发电站，费用为v（发电站的输出功率可以供给任意多个矿井）。

将这口矿井与另外的已经有电力供应的矿井之间建立电网，费用为p。

小C 希望身为「NewBe_One」计划首席工程师的你帮他想出一个保证所有矿井电力供应的最小花费。

输入

第一行一个整数n，表示矿井总数。

第2∼n+1行，每行一个整数，第i个数vi表示在第i口矿井上建立发电站的费用。

接下来为一个n×n的矩阵p，其中pij表示在第i口矿井和第j口矿井之间建立电网的费用（数据保证有pij=pji，且pii=0)。

输出

输出仅一个整数，表示让所有矿井获得充足电能的最小花费。

solution:

开始想到的是这样的一个贪心：

考虑用Kruskal先求出最小生成树。
然后对于生成树的每一个节点。
考虑和它相邻的边。
如果所有和它相连的边都比它的值大。
那么建立电站。
否则连边。
但是png里构造的数据可以卡掉这个不成熟的贪心

如你所见，这个贪心被卡爆了

如果最小生成树是这样子的：

(5)–6–(7)–10–(1)

那么就死透了（可以自己模拟一下，按照那个贪心出来是13，但是有一个12的答案）。

那么这里有一个神奇的做法。

因为我们的贪心被卡掉就是因为在点上的元素不好搞。

都说这是图论图论，mst，重点就是在边的处理上。

干脆我们就把这个点值化成边值。

也就是在原图上新加一个点（超级源点）

它和 $1-n$ 的节点全部相连。

边权是在 $i$ 建立电站的cost $v_i$

也就相当于我们把每一个点化成边来处理.

然后在原图上跑一遍最小生成树就好了。

这个思想挺妙的，开始一直没想出来（还是看了题解才知道的）。

【国家集训队2012】[Tree I]

description:

描述

原题来自：2012 年国家集训队互测
给你一个无向带权连通图，每条边是黑色或白色。让你求一棵最小权的恰好有 need 条白色边的生成树。题目保证有解。

输入
第一行 V,E,need 分别表示点数，边数和需要的白色边数。
接下来 E 行，每行 s,t,c,col 表示这边的端点（点从 0 开始标号），边权，颜色（0 白色，1 黑色）。

输出
一行表示所求生成树的边权和。

solution:

考虑我们直接给原图跑一遍最小生成树。

无非就是三种情况：

  1. 刚好有$need$条白边
  2. 比$need$少了一些
  3. 比$need$多了一些

对于1我们不用干啥。

然后考虑，怎么样在求最小生成树的同时增减白边的条数呢？

这里有一种神奇的想法。

我们给所有白边都加上一个值$val$

那么有可能加到某个值，一条白边就被加入（踢出）mst了。

怎么确定这个值？

简单思考一下发现 $val$ 是满足单调性的。

所以考虑二分答案。

如果求出来的真实白边数 $real$ 大于$need$ ，考虑就要增加 $val$ 来减少白边。

反之亦然。

然后在$need \leq real$ 的时候更新答案就行了（因为这个时候才是可行的，可以自己证明一下。

这里给出代码：


#include<bits/stdc++.h>
using namespace std;

const int si=2e5+10;
int n,m,need;
int pa[si];

struct node{
	int x,y,val,ty;
	bool operator < (const node &u)const{
		if(val==u.val) return ty<u.ty;
		return val<u.val;
	}
}e[si];
int root(int x){
	if(pa[x]!=x){
		pa[x]=root(pa[x]);
	}
	return pa[x];
}
bool Union(int x,int y){
	int rx=root(x),ry=root(y);
	if(rx==ry){
		return false;
	}
	pa[rx]=ry;
	return true;
}

void kruskal(int &sum,int &real){
	sort(e+1,e+1+m);
	for(register int i=1;i<=n;++i){
		pa[i]=i;
	}
	for(register int i=1;i<=m;++i){
		if(Union(e[i].x,e[i].y)){
			sum+=e[i].val;
			real+=(int)(e[i].ty==0);
		}
	}
	return;
}

int main(){
	cin>>n>>m>>need;
	for(register int i=1;i<=m;++i){
		cin>>e[i].x>>e[i].y>>e[i].val>>e[i].ty;
		++e[i].x,++e[i].y;
	}
	int l=-114,r=114;
	int res=0;
	while(l<=r){
		int mid=(l+r)>>1;
		for(register int i=1;i<=m;++i){
			if(e[i].ty==0) e[i].val+=mid;
		}
		int sum=0,real=0;
		kruskal(sum,real);
		if(real>=need) res=sum-(need*mid),l=mid+1;
		else{
			r=mid-1;
		}
		for(register int i=1;i<=m;++i){
			if(e[i].ty==0) e[i].val-=mid;
		}
	}
	printf("%d\n",res);
	return 0;
}

P1522 [USACO2.4]牛的旅行 Cow Tours

description

描述
农民 John 的农场里有很多牧区。有的路径连接一些特定的牧区。

一片所有连通的牧区称为一个牧场。但是就目前而言，你能看到至少有两个牧区不连通。

现在，John 想在农场里添加一条路径 ( 注意，恰好一条 )。对这条路径有这样的限制：

一个牧场的直径就是牧场中最远的两个牧区的距离 ( 本题中所提到的所有距离指的都是最短的距离 )。

考虑如下的两个牧场，图１是有 5 个牧区的牧场，牧区用“*”表示，路径用直线表示。

每一个牧区都有自己的坐标：
图１所示的牧场的直径大约是 12.07106, 最远的两个牧区是 A 和 E，它们之间的最短路径是 A-B-E。

这两个牧场都在 John 的农场上。John 将会在两个牧场中各选一个牧区，

然后用一条路径连起来，使得连通后这个新的更大的牧场有最小的直径。

注意，如果两条路径中途相交，我们不认为它们是连通的。只有两条路径在同一个牧区相交，

我们才认为它们是连通的。
现在请你编程找出一条连接两个不同牧场的路径，使得连上这条路径后，这个更大的新牧场有最小的直径。 

输入第 1 行：一个整数 N (1 ≤ N ≤ 150), 表示牧区数；

第 2 到 N+1 行：每行两个整数 X，Y ( 0 ≤ X，Y≤ 100000 )， 表示 N 个牧
区的坐标。每个牧区的坐标都是不一样的。

第 N+2 行到第 2*N+1 行：每行包括 N 个数字 ( 0 或 1 ) 表示一个对称邻接矩阵。

例如，题目描述中的两个牧场的矩阵描述如下：
A B C D E F G H
A 0 1 0 0 0 0 0 0
B 1 0 1 1 1 0 0 0
C 0 1 0 0 1 0 0 0
D 0 1 0 0 1 0 0 0
E 0 1 1 1 0 0 0 0
F 0 0 0 0 0 0 1 0
G 0 0 0 0 0 1 0 1
H 0 0 0 0 0 0 1 0
输入数据中至少包括两个不连通的牧区。 
输出只有一行，包括一个实数，表示所求答案。数字保留六位小数。

solution

考虑先对于每个牧区求相互之间的最短路。

然后对于每一个牧区求和它联通的点最远的距离 D[i]

再对于每个牧场更新直径。

然后枚举所有不连通的点连边之后的情况然后取最值。

值得注意的是连了之后可能还不如原来的大，所以要在最后再取一次最值。

  • code:

#include<bits/stdc++.h>
using namespace std;

const int inf=114514333;
const int si=500;
int n;
double f[si][si];
struct node{
	int x,y;
}p[si];
double clac(int u,int v){
	return (double)sqrt((p[u].x-p[v].x)*(p[u].x-p[v].x)+(p[u].y-p[v].y)*(p[u].y-p[v].y));
}

void Floyd(){
	for(register int k=1;k<=n;++k){
		for(register int i=1;i<=n;++i){
			for(register int j=1;j<=n;++j){
				f[i][j]=min(f[i][j],f[i][k]+f[k][j]);
			}
		}
	}
}

double D[si];//直径（可以看做对于每个点的） 
double D1,D2=inf;
int main(){
	cin>>n;
	for(register int i=1;i<=n;++i){
		cin>>p[i].x>>p[i].y;
	}
	for(register int i=1;i<=n;++i){
		for(register int j=1,o;j<=n;++j){
			scanf("%1d",&o);
			if(o==1) f[i][j]=clac(i,j);
			else if(i!=j) f[i][j]=(double)inf;
		}
	}
	Floyd();
	for(register int i=1;i<=n;++i){
		for(register int j=1;j<=n;++j){
			if(f[i][j]!=inf){
				D[i]=max(D[i],f[i][j]);
			}
			D1=max(D1,D[i]);
		}
	}
	for(register int i=1;i<=n;++i){
		for(register int j=1;j<=n;++j){
			if(f[i][j]==inf){
				double w=clac(i,j);
				D2=min(D2,D[i]+w+D[j]);
			}
		}
	}
	double ans=max(D1,D2); 
	printf("%.6lf\n",ans);
	return 0;
}

P1576 最小花费

description

在n个人中，某些人的银行账号之间可以互相转账。这些人之间转账的手续费各不相同。给定这些人之间转账时需要从转账金额里扣除百分之几的手续费，请问A最少需要多少钱使得转账后B收到100元。

输入

第一行输入两个正整数n,m，分别表示总人数和可以互相转账的人的对数。

以下m行每行输入三个正整数x,y,z，表示标号为x的人和标号为y的人之间互相转账需要扣除z%的手续费 (z<100)。

最后一行输入两个正整数A,B。数据保证A与B之间可以直接或间接地转账。

输出

输出A使得B到账100元最少需要的总费用。精确到小数点后8位。

solution

其实这题不用跑最长路，直接从终点反过来跑最短路就好了。

然后改一下松弛：double newd=(double)dist[u]/(1-0.01*w);，

dist[B]初始化成 $100$ 就行。

P5767 [NOI1997] 最优乘车

description

描述

H城是一个旅游胜地，每年都有成千上万的人前来观光。为方便游客，巴士公司在各个旅游景点及宾馆，饭店等地都设置了巴士站并开通了一些单程巴士线路。每条单程巴士线路从某个巴士站出发，依次途经若干个巴士站，最终到达终点巴士站。

一名旅客最近到H城旅游，他很想去S公园游玩，但如果从他所在的饭店没有一路巴士可以直接到达S公园，则他可能要先乘某一路巴士坐几站，再下来换乘同一站台的另一路巴士, 这样换乘几次后到达S公园。

现在用整数1,2,…N 给H城的所有的巴士站编号，约定这名旅客所在饭店的巴士站编号为1，S公园巴士站的编号为N。

写一个程序，帮助这名旅客寻找一个最优乘车方案,使他在从饭店乘车到S公园的过程中换车的次数最少。

输入

第一行有两个数字M和N(1≤M≤100 1<N≤500），表示开通了M条单程巴士线路，总共有N个车站。从第二行到第M行依次给出了第1条到第M条巴士线路的信息。其中第i+1行给出的是第i条巴士线路的信息，从左至右按运行顺序依次给出了该线路上的所有站号相邻两个站号之间用一个空格隔开。

输出

只有一行。如果无法乘巴士从饭店到达S公园，则输出"NO"，否则输出你的程序所找到的最少换车次数，换车次数为0表示不需换车即可到达。

solution

注意到这是无权图，而且SSSP不好做判断，所以直接写BFS。

很简单所扔code：

#include<bits/stdc++.h>
using namespace std;

const int si=500+21;
int g[si][si];
int a[si];
struct node{
    int u,dep;
}q[si];
bool vis[si];
int n,m;
int bfs(int s){
	memset(vis,0,sizeof(vis));
	q[0]=(node){s,0};
	vis[s]=1;
	int head=0,tail=1;
	while(head<tail){
		node p=q[head++];
		if(p.u==n)return p.dep-1;
		int u=p.u;
		for(register int v=1;v<=si;v++){
			if(g[u][v]&&vis[v]==0){
				q[tail++]=(node){v,p.dep+1};
				vis[v]=1;
			}
		}
	}
	return -1;
}
int main(){
	scanf("%d%d",&m,&n);
	for(register int p=1;p<=m;++p){
		int cnt=0;
		scanf("%d",&a[++cnt]);
		char ch=getchar();
		while(ch==' '){
			scanf("%d",&a[++cnt]);
			ch=getchar();
		}
		for(register int i=1;i<=cnt;i++){
            for(register int j=i+1;j<=cnt;j++){
                g[a[i]][a[j]]=1;
            }
        }
			
	}
	int ans=bfs(1);
	if(ans==-1) cout<<"NO"<<endl;
	else cout<<ans<<endl;
	return 0;
}

P1220 关路灯

description

   <h2>题目描述</h2>
    <div>某一村庄在一条路线上安装了 $n$ 盏路灯，每盏灯的功率有大有小（即同一段时间内消耗的电量有多有少）。老张就住在这条路中间某一路灯旁，他有一项工作就是每天早上天亮时一盏一盏地关掉这些路灯。

为了给村里节省电费，老张记录下了每盏路灯的位置和功率，他每次关灯时也都是尽快地去关，但是老张不知道怎样去关灯才能够最节省电。他每天都是在天亮时首先关掉自己所处位置的路灯，然后可以向左也可以向右去关灯。开始他以为先算一下左边路灯的总功率再算一下右边路灯的总功率，然后选择先关掉功率大的一边，再回过头来关掉另一边的路灯，而事实并非如此，因为在关的过程中适当地调头有可能会更省一些。

现在已知老张走的速度为 $1m/s$，每个路灯的位置（是一个整数，即距路线起点的距离，单位：$m$）、功率（$W$），老张关灯所用的时间很短而可以忽略不计。

请你为老张编一程序来安排关灯的顺序，使从老张开始关灯时刻算起所有灯消耗电最少（灯关掉后便不再消耗电了）。</div>

    <h2>输入输出格式</h2>
    <h3>输入格式</h3>
    <br />
    <div>第一行是两个数字 $n$（表示路灯的总数）和 $c$（老张所处位置的路灯号）；

接下来 $n$ 行，每行两个数据，表示第 $1$ 盏到第 $n$ 盏路灯的位置和功率。数据保证路灯位置单调递增。</div>
    <h3>输出格式</h3>
    <br />
    <div>一个数据，即最少的功耗（单位：$J$，$1J=1W\times s$）。</div>

    <h2>输入输出样例</h2>
                        <h3>输入样例 #1</h3>
            <pre><code>5 3
2 10
3 20
5 20
6 30
8 10</code></pre>
            <h3>输出样例 #1</h3>
            <pre><code>270  </code></pre>
            
            <h2>说明</h2>
        <div>### 样例解释

此时关灯顺序为 `3 4 2 1 5`。

### 数据范围

$1\le n\le50$，$1\le c\le n$。</div>

solution

这个题是一个比较典型的 小区间推大区间。

因为你会发现老张走路不一定是一条线。

也就是说正常的 线性$\text{dp}$ 是没有办法做这种题的（不能按从前往后或者从后往前推）。

那么这个时候就要联想到 区间$\text{dp}$ 这个“不按线性顺序”更新的东西

注意到这里老张是想“关灯”且尽量早关 , 也就是说他如果经过了一个灯他必定会关掉，不然再折回来就会浪费，不是最优的。

也就是说，他走路的顺序一定是类似于这种的（直线也包括在内）：

那么也就是说，对于一个区间,老张如果站在这个区间的端点的时候，那么他必定早已关完了这个区间所有的灯。

所以我们就可以从 $f[start][start]$ 这个最小的区间慢慢推到最大的 $f[1][n]$ 这个区间。

然后一个区间是有两个端点的，所以要用 $[0/1]$ 区分一下（因为不同的端点可能有不同的值，也就是方向的不同导致了结果的不同）。

至于怎么转移呢？

我们这里完全没有必要用一个 $k$ 去枚举上一个决策点了，因为我们推出来了这一条：

他如果经过了一个灯,那么他必定会关掉。

所以，比如在区间左端点的时候，那么就可以直接从 $l+1$ 这个位置转移过来。

所以每一次推大区间的时候都是 每次$+1$ 这样子推过去的。

当然你转移的时候也要考虑这个小区间是在左端点过来还是右端点过来。

（你也看到了，上图的路径是有重复的，所以不一定是离他最近的端点转移过来的）

然后考虑怎么计算花费，这里直接计算一下小区间的功耗然后加上扩展出来的（也就是端点的花费）就好了。

代码：

#include<bits/stdc++.h>
using namespace std;

const int si=60;
int a[si],b[si],sum[si];
int n,m,root;
int f[si][si][2];

int main(){
	scanf("%d%d",&n,&root);
	memset(f,127,sizeof(f));
	for(register int i=1; i<=n;i++){
		scanf("%d%d",&a[i],&b[i]);
		sum[i]=sum[i-1]+b[i];
	}
	f[root][root][0]=f[root][root][1]=0;
	for(register int l=2;l<=n;l++){
		for(register int i=1;i+l-1<=n;i++){
			register int j=i+l-1;
			f[i][j][0]=min(f[i+1][j][0]+(a[i+1]-a[i])*(sum[i]+sum[n]-sum[j]),f[i+1][j][1]+(a[j]-a[i])*(sum[i]+sum[n]-sum[j]));
			f[i][j][1]=min(f[i][j-1][0]+(a[j]-a[i])*(sum[i-1]+sum[n]-sum[j-1]),f[i][j-1][1]+(a[j]-a[j-1])*(sum[i-1]+sum[n]-sum[j-1]));
		}
	}
	int ans=min(f[1][n][0],f[1][n][1]);
	printf("%d",ans);
	return 0;
}

P2339 [USACO04OPEN]Turning in Homework G

description

    <h2>题目描述</h2>
    <div>贝茜有 $ C $ ( $ 1 \leq C \leq 1000 $ )门科目的作业要上交，之后她要去坐巴士和奶牛同学回家。

每门科目的老师所在的教室排列在一条长为 $ H $ ( $ 1 \leq H \leq 1000 $ )的走廊上，他们只在课后接收作业，交作业不需要时间。贝茜现在在位置0，她会告诉你每个教室所在的位置，以及走廊出口的位置。她每走1个单位的路程，就要用1秒。她希望你计算最快多久以后她能交完作业并到达出口。</div>

    <h2>输入输出格式</h2>
    <h3>输入格式</h3>
    <br />
    <div>第一行：三个整数 $C$，$H$ 和 $B$，$1 \le C \le 1000,1 \le H \le 1000,0 \le B \le H$。

第二行到 $C+1$ 行：第 $i+1$ 行有两个整数 $X_i$ 和 $T_i$，$0 \le X_i \le H,0 \le T_i \le 10000$。

 $B$ 表示车站位置。

 $X_i$ 表示第  $i$ 份作业应该在  $X_i$ 这个位置交。

 $T_i$ 表示这个位置（教室）的这个科目老师（也就是收这份作业的老师）下课的时间。</div>
    <h3>输出格式</h3>
    <br />
    <div>单个整数，表示贝西交完作业后走到车站的最短时间
</div>

    <h2>输入输出样例</h2>
                        <h3>输入样例 #1</h3>
            <pre><code>4 10 3
8 9
4 21
3 16
8 12
</code></pre>
            <h3>输出样例 #1</h3>
            <pre><code>22</code></pre>
            
            <h2>说明</h2>
        <div>走到坐标 8 处，第 9 分钟交一本作业，等到第 12 分钟时，交另一本作业。再走到坐标 4 处交作业，最后走到坐标 3 处，交最后一本作业，此地就是车站所在位置，共用时 22 分钟
</div>

（这道题luogu上缺了题面还是我给补的lol）

solution

这里你会发现和上一道关路灯基本是一样的。

也就是说Bessie的行走路线也是折线型不按顺序来的。

所以我们又一次地想到了区间 $\text{dp}$

但是这里有限制条件啊，咋办？

我们肯定没法用上一道关路灯的办法来设计状态。

因为Bessie经过一个教室不一定会交作业（因为还没下课）

那么我们就可以换个方式。

我们不是从小区间推大区间了。

我们从大区间推小区间。

也就是考虑 $f[i][j][0/1]$ 表示除了这一个区间以外，作业都交完了，马上去交i或者j处的作业的最值。

但是你会发现时间这东西烦人的很，会干扰我们的转移，比如出极大的数据（

所以干脆先排个序先，因为我们基本按照时间去DP肯定比之前的乱DP更优。

那么我们用时间排序之后，就可以利用现在时间单调的特性。

从 $f[1][n]$ 推到 每一个$f[i][i]$，然后和巴士站比较一下距离取最值就好了。

代码：

#include<bits/stdc++.h>
using namespace std;

const int si=2e3+10;
int n,h,b;
int f[si][si][2];

struct node{
	int x,t;
	bool operator <(const node &p)const{
		return x<p.x;
	}
}a[si];
int cald(int u,int v){
	return abs(a[u].x-a[v].x);
}

int main(){
	cin>>n>>h>>b;
	for(register int i=1;i<=n;i++){
		cin>>a[i].x>>a[i].t;
	}
	memset(f,0x3f3f3f,sizeof f);
	sort(a+1,a+n+1);//排序，贪心
	f[1][n][0]=max(a[1].x,a[1].t),f[1][n][1]=max(a[n].x,a[n].t);
	for(register int i=1;i<=n;i++){
		for(register int j=n;j>=1;j--){//倒着决策来从小推大
			f[i][j][0]=min(f[i][j][0],max(f[i-1][j][0]+cald(i-1,i),a[i].t)),f[i][j][0]=min(f[i][j][0],max(f[i][j+1][1]+cald(j+1,i),a[i].t));
			f[i][j][1]=min(f[i][j][1],max(f[i-1][j][0]+cald(i-1,j),a[j].t)),f[i][j][1]=min(f[i][j][1],max(f[i][j+1][1]+cald(j+1,j),a[j].t));
		}
	}
	int res=0x3f3f3f3f;
	for(register int i=1;i<=n;i++){
		res=min(res,
		min(f[i][i][0]+abs(b-a[i].x),
		f[i][i][1]+abs(b-a[i].x)));
	}//取最值
	cout<<res<<endl;
	return 0;
 }
