题解 UVA515 【King】

一道适合差分约束入门的题。

建议把难度改绿。

看到题面里有这么一句话：

有 $n$ 个数，$m$个限制条件。

然后又看到了 形如 $something +something < something$的柿子

所以我们应该能十分明显的想到差分约束了。

但是一个比较棘手的问题。

题目中给的是一个序列里的和。

而我们的差分约束里面的限制条件一般都是
两个数之和小于或者大于某个数的不等式形式。

所以我们就需要转化一下。

再好好读一下题，不难发现这些数是在一个序列里连续的出现的。

所以我们可以用一个 sum[] 数组维护一下前缀和来处理（当然可以这么理解，我的代码里没有直接用 sum[] ,只是用了一下它的思想）。

然后我们建图的时候直接这么做就可以了。

es[i].from=a+b+1;
  es[i].to=a;
  es[i].dist=-c-1;//gt

（直接这么用 $a,b$ 是因为他给出的$a,b$ 是下标形式）

然后操作符是 lt 的时候同理就行。

最后就可以愉快的判断负环然后输出啦！

代码如下：

struct edge{
    int from,to,dist;
}es[100+10];

int dist[100+10];

bool bellman_ford(int n,int m){
    for(register int i=1;i<=n;++i){
        dist[i]=0;
    } 
    for(register int i=1;i<=n;++i){
        for(register int j=0;j<m;++j){
            edge &e=es[j];
            if(dist[e.to]>dist[e.from]+e.dist){
                dist[e.to]=dist[e.from]+e.dist;
            }
        }
    }
    for(register int j=0;j<m;++j){
        edge &e=es[j];
        if(dist[e.to]>dist[e.from]+e.dist){
            return false;
        }
    }
    return true;
}

int n,m;
int a,b,c;

int main(){
    char op[3];
    while(scanf("%d",&n),n){
        scanf("%d",&m);
        for(register int i=0;i<m;++i){
            scanf("%d%d%s%d",&a,&b,str,&c);
            if(op[0]=='g'){
                es[i].from=a+b+1;
                es[i].to=a;
                es[i].dist=-c-1;
            }
            else{
                es[i].from=a;
                es[i].to=a+b+1;
                es[i].dist=c-1;
            }
        }
        if(bellman_ford(n,m)){
            puts("lamentable kingdom");
        } 
        else{
            puts("successful conspiracy");
        } 
    }
    return 0;
}
